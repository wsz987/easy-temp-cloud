# 鉴权设计

## 目标

为上传页面、上传 API 和本地文件下载增加基于环境变量短密码的鉴权；不引入数据库或持久化会话。

## 配置与密钥生命周期

- 新增必填环境变量 `AUTH_PASSWORD`。它是部署者可记忆的短密码，长度限制为 6 到 16 个 ASCII 字符。服务在启动时拒绝空值或不符合长度限制的配置。
- 每次服务启动生成新的 32 字节内存签名密钥。它不写入磁盘。
- 重启会使已有浏览器会话和文件分享链接立即失效；修改 `AUTH_PASSWORD` 后重启同样生效。

## 浏览器登录

- 未认证的 `GET /` 返回一个只含密码表单的登录页面。
- `POST /login` 接收密码，使用常量时间比较校验 `AUTH_PASSWORD`。成功后返回到 `/`，并设置 7 天有效的 `HttpOnly`、`SameSite=Lax` 会话 Cookie。
- Cookie 内容含过期时间，并由本进程启动时的内存签名密钥签名。服务不保存会话记录。
- `POST /logout` 清除 Cookie，并返回登录页。
- 已登录的浏览器可访问上传页、`/api/config` 和标准 tus 上传端点；登录页所需静态资源保持公开。

## API 与文件分享

- 外部单请求上传使用 `POST /api/upload?pwd=<AUTH_PASSWORD>`。`pwd` 仅用于该端点，作为 bearer credential 处理。
- 每个上传成功响应都返回由进程内签名密钥、文件 SHA-256 ID 生成的文件专用 URL：`/files/{id}?key=<signature>`。
- `key` 只对其指定文件有效，不能登录、上传或访问其他文件。它由启动期密钥计算，因此重启后会失效。
- 本地存储的 `GET /files/{id}` 必须附带有效的文件专用 `key`。

## 存储和公开端点

- `GET /healthz` 保持匿名，用于 Docker 健康检查。
- OSS 仍使用公开 Bucket URL，无法由本服务在下载时鉴权；OSS 上传响应保持现有公开对象 URL 行为。本次仅保证本地存储下载链接带 `key`。

## 失败处理与保护

- 浏览器登录失败、缺失或无效 Cookie，以及无效分享 `key`，统一返回 `401 Unauthorized`，不泄露密码是否接近正确值。
- `/api/upload` 缺失或错误 `pwd` 返回 `401 Unauthorized`，且必须在读取或预留上传请求体前返回。
- 内存限速器按客户端 IP 记录失败尝试，在固定窗口内超过阈值时返回 `429 Too Many Requests`。进程重启清空该记录。

## 验收条件

- 未登录无法取得上传页面、客户端配置或 tus 上传能力。
- 正确密码可登录，且 Cookie 可用于网页 tus 上传；错误密码无法登录。
- `/api/upload` 只有携带正确 `pwd` 时可上传。
- 本地文件仅能通过返回的专用分享链接读取，篡改 ID 或 `key` 均失败。
- 服务重启后旧 Cookie 与旧本地分享链接失效，新登录和新上传链接可正常使用。
